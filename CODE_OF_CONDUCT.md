# Code of Conduct

We want **Browser Hardware Diagnostics** to be a respectful, welcoming, and technically focused open-source project.

## Our standards

Contributors are expected to:

- be respectful in issues, pull requests, and reviews
- assume good intent while discussing technical tradeoffs
- welcome contributors from different backgrounds and skill levels
- give actionable, technical feedback instead of personal criticism
- keep discussions focused on improving the shared toolkit

## Unacceptable behavior

The following behavior is not acceptable:

- harassment or intimidation
- personal attacks or insulting language
- discriminatory or exclusionary behavior
- bad-faith trolling or repetitive disruption
- publishing private information without permission

## Scope

This code of conduct applies to:

- issues
- pull requests
- code review discussions
- repository discussions and related project spaces

## Reporting

If you need to report a problem, contact the maintainer through the project entry point:

- **Maintainer:** https://github.com/OneSpiral

When reporting an issue, include enough context for the maintainers to review it fairly.

## Enforcement

Maintainers may remove, edit, or reject contributions and interactions that violate this code of conduct in order to keep the project respectful and welcoming.
