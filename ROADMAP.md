# Roadmap

## Goal

Build the best open-source browser hardware diagnostics toolkit, with community contributors helping expand technical capabilities over time.

## Near term

- strengthen shared diagnostics primitives
- improve documentation for adopters and contributors
- grow the next contributor-ready public backlog wave

## Current public issue wave

The next public backlog wave focuses on reusable technical improvements that fit the open-source toolkit boundary:

- gamepad haptics and vibration capability diagnostics
- locale-aware keyboard legends and layout caveats
- mouse drag / hold diagnostics primitives
- monitor motion and ghosting utilities
- reusable browser API fixtures and mocks for diagnostics tests

## Diagnostics roadmap

### Gamepad
- multi-controller support
- vibration / haptics abstraction
- better drift visualization
- calibration helpers

### Keyboard
- improved rollover diagnostics
- layout abstraction for more locales
- typing diagnostics primitives

### Mouse
- better polling-rate estimation
- drag / hold diagnostics
- richer click timing analysis

### Monitor
- more fullscreen test patterns
- motion / ghosting utilities
- better display capability detection

### Audio
- richer microphone quality diagnostics
- safer speaker test presets
- improved visualization primitives

## Ecosystem roadmap

- adapter points for branded adopters
- clearer extension interfaces
- more reusable technical modules
- better test fixtures and mocks
- contributor-friendly issue breakdowns

## Explicitly separate

The roadmap above is for the open-source technical solution.

Private growth systems, branded content, and distribution strategy are intentionally outside this roadmap.
